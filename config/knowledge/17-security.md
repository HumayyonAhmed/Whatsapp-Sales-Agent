# Security

## What's true and safe to say

- AquaFlow is a cloud-based system, hosted on Supabase, a widely-used cloud
  backend platform.
- Data is transmitted over encrypted HTTPS connections.
- Access is role-based: an Owner, Delivery Boy, and Customer each only see
  the data relevant to their role — a driver cannot see the Owner's full
  business data, and a customer can only see their own account.
- Regular backups are part of running a cloud-hosted system on
  infrastructure like Supabase.

## What NOT to claim

Do not state or imply any of the following unless a team member confirms it
directly, since these are not documented facts to give confidently:

- Specific compliance certifications (e.g. ISO, SOC 2, PCI-DSS)
- Specific encryption algorithms or key lengths
- Specific data residency/location guarantees
- Formal security audit results

## How to handle deeper security questions

If a prospect (especially a larger commercial customer) asks detailed
security/compliance questions beyond what's listed above as safe to say,
the right response is to be honest that a team member will follow up with
the specific details, rather than guessing or improvising an answer that
could be wrong. This is especially important for security topics — an
inaccurate claim here could create real problems for the customer
relationship and for AquaFlow's credibility.
